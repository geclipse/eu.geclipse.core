AWS middleware prerequisites

The g-Eclipse framework provides seamless access to Cloud infrastructures of Amazon Web Services (AWS). AWS contains a suite of services for building applications, were the g-Eclipse AWS implementation currently support the Simple Storage Service (S3) and the Elastic Cloud Computing (EC2).

Access to S3 and EC2 resources is based on a valid user account at Amazon. In order to use g-Eclipse with the Cloud services from Amazon the following requisites must be fulfilled.

  • A g-Eclipse user needs an valid general account at Amazon. That is the same account that can be used for non-Cloud services as AWS.
  • The user must register himself for the S3 and EC2 services at Amazon Cloud Webservice (AWS). Find more information here.
  • In order to use the AWS systems, the user must know his Access Identifiers. Find them under "Your Account --> Access Identifiers" on the AWS web page after your login. You need to know your "Access Key ID" and the "Secret Access Key".

When the above prerequisites are fulfilled, you can start using AWS Cloud services with g-Eclipse.
