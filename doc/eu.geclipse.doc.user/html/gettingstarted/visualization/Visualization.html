Scientific Data Visualisation using g-Eclipse

Introduction To Visualisation in g-Eclipse

This is an introduction page describing the Visualisation module of g-Eclipse. Note that this work is in progress and more information is being added periodically.

If you want to visualise scientific data sets computed on the Grid, you don't have to leave g-Eclipse. The g-Eclipse project integrates a visualisation tool, which, in its final state, will allow you to analyse your data sets applying the current data extraction algorithms from the well known open source visualisation software system - Visualisation Toolkit (VTK) from Kitware.

The goal of the g-Eclipse visualisation module is to allow local as well as remote rendering of data sets that may be stored locally or remotely. In both cases, g-Eclipse is used to create a VTK Pipeline file, where one or more data extraction algorithms are fully defined together with the location of the data set, which the algorithms shall be applied to.

This chapter is split into four main sections. The steps of creating VTK Pipeline File in g-Eclipse are covered in the first section. How to fill in the content of the VTK Pipeline file is the topic of the second section. The third section contains instructions for running the visualisation on a local machine and, finally, the same is described for remote machine rendering.
