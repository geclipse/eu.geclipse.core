C/C++ MPI Grid Application Launch Configuration Dialog

Always consider that the application will run on a remote machine and therefore the following requirements have to be met.

  • The dependencies of the application have to be available remotely.
  • The remote environment has to be set up correctly.
  • GLogin has to be available on the remote host.
  • The project has to be built for the remote architecture and libraries.

The Main Tab

Allows to select the project and a executable which will be used to run the application. Moreover the the program's in- and output can be connected to a console.




The Arguments Tab

Provides fields to specify program arguments and the program's working directory.




The Grid Tab

The application will be started on a remote machine defined by a contact string. Optionally it is possible to specify the remote GLogin executable instead of using the default one. Furthermore the encryption of the communication to the remote host can be deactivated.




The MPI Tab

This tab offers a field to set the number of processes which will be used to run the application. To use another than the default mpi implementation of the remote machine the path to the mpirun/mpiexec executable can be specified.




The Debug Tab

There are two ways of remote debugging are available.

  • gdb + nc - uses netcat and gdb on the remote machine.
  • gdbserver - uses gdbserver on the remote machine.
The remote debugging uses ports forwarded through GLogin which can be defined using the fields of this tab. A script is used to distribute gdb/gdbserver among the processes which currently only works on some mpi implementations. It is possible to automatically stop at the main method of the application.




The Staging Tab

Currently GridFTP is used to transfer the executable file, selected in the Main Dialog, to the computing element from the contact string specified in the Grid Tab. In addition to enabling or disabling of the executable staging it is possible to set the port used for GridFTP.




The Source Tab

The source search path for the debugger can be specified in this tab.
