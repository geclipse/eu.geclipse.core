g-Eclipse - Authentication Tokens

To create an authentication token, first select the Authentication Tokens view from the lower part of the Eclipse window. Then click on the new icon in the top right of this view



and you will be presented with the following window.



You may choose to create a Grid proxy or a VOMS proxy for the token type.

Grid proxy
Click next and you will be presented with the following dialog box.



Specify the user certificate, the user key and the key passphrase. By default your proxy will be valid for one day since activation. You may change this by specifying different proxy lifetime. Finally, press finish and your proxy will be validated and will appear as an entry in the Authentication Tokens view. To activate this token press the activation button on the top right of this view.

VOMS proxy
Not yet implemented.

Note on creating authentication tokens:
If you decide to submit a new job without first creating your authentication token, you will be notified by the application and offered that a token be created. Accepting this will take you directly to the grid proxy dialog where you can complete the token creation. The new job will be started only after successful creation of your token.
