g-Eclipse - Known Issues

  • Firewalls:

    g-Eclipse has the same connectivity requirements as any other client software for a specific grid middleware. Therefore you will need to be able to establish outgoing TCP connections to a series of ports, and in a few cases also allow some incoming open ports. Check the middleware specific documentation for more details.

  • Changes in Job Description are not reflected immediately in Properties view:

    If you modify a JSDL file, then the Properties view will still show old values for the job description. You have to restart g-Eclipse to see the updated values. Despite this, the modified (correct) job description will be used for job submission even without having to restart g-Eclipse.

  • ProcessStatusView only shows anonymous pid-lists:

    The current implementation of the Process Status View started with the Action: "Monitor Computing Resource" will only display anonymous pid-lists of the remote resources as long as the GridFTP connection to the remote host is successful. Full functionality is going to be available for the 1.0.0 Release.

  • Known Issues with Mac OS X:

    Sometimes, collapsing the tree in the Grid Project View or the Glue Information Viewer will either not redraw the tree, or, in some cases, cause the application to crash.
    VO Management:
    Web View using HTTPS does not work on Mac OS X.

  • JSDL Editor Issues:

    Input field validations:
    The current implementation of the JSDL editor does not provide input field validation therefore, the user must provide correct input as per the JSDL Specification v.1.0. Input guidelines are given on the help pages of the respective JSDL editor pages.
    File System and Additional Resources Sections Functionality:
    The current implementation of the JSDL editor provides limited functionality for manipulating the elements under the File System and Additional Resources Elements sections of the Resources Page.

  • Workflow Builder

    The workflow builder is sometimes unstable and it does not yet have the full functionality to allow submission to a grid middleware.
