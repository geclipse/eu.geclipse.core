What's new in g-Eclipse

  • Initial...
    • Create a job description using a simple wizard or the JSDL editor
    • Submit your job to a gLite resource from the Eclipse workbench
    • Create connections from your local machine to remote storage resources on the Grid and copy files between them
    • Easy to use Authentication Tokens Management
    • Information system to query available resources on the Grid
    • Shell-like access to remote Grid resources from within the Eclipse WorkBench
    • ... and much more
  • g-Eclipse 1.0 M2
    • First support for the GRIA middleware
    • New import functionality for VOMS VOs
    • ... and of course a lot of bug fixes
  • g-Eclipse 1.0 M3
    • First support for Amazon's S3
    • Full support for new problem reporting mechanism
    • ... and of course a lot of bug fixes
  • g-Eclipse 1.0 M4 This release is thought to be a maintenance release containing mainly quality work.
    • Already submitted jobs can now be resubmitted if they have failed
    • UI enhancements and improvements
    • Code cleanup
    • User documentation improvements
    • Cheatsheets fixed and improved
    • ... and of course a lot of bug fixes
  • g-Eclipse 1.0 M5
    • Local input files are now sent to the GRIA during job submission
    • When user delete job from local workspace, he has also possibility to remove that job also from the Grid
    • During job submission user may select target folder for submitted job and change job name
    • Job Details view show GRIA specific information about job
    • New GUI for the Batch Editor
    • Improved GRIA Site Administrator Editor
    • Code cleanup
    • ... and of course a lot of bug fixes
